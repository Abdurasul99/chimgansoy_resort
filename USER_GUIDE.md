# USER_GUIDE.md — CHIMGAN DARBAZA (простыми словами)

Короткое руководство для владельца/менеджера. Технические детали — в
[TECHNICAL_DOCS.md](./TECHNICAL_DOCS.md), доступы — в `ACCESS.md`.

> Важно: сайт сделан на **Next.js** и живёт на **Vercel** (не на Cloudflare
> Pages). Бронирование — через **Exely**. Это на случай, если где-то писали
> иначе.

---

## 1. Как скачать сайт себе (пошагово)

1. Установите **Git** (git-scm.com) и **Node.js 22** (nodejs.org).
2. Откройте терминал (на Windows — «Command Prompt» или «PowerShell») и введите:
   ```
   git clone https://github.com/Abdurasul99/chimgansoy_resort.git
   cd chimgansoy_resort
   npm install
   ```
3. Попросите текущего разработчика прислать файл **`.env.local`** (там ключи —
   в Git его нет намеренно) и положите его в папку проекта.
4. Запуск на своём компьютере:
   ```
   npm run dev
   ```
   Откройте в браузере `http://localhost:3000`.

Ничего править руками для этого не нужно — сайт запустится как есть.

---

## 2. Как внести простые правки самому

Весь текст, цены, контакты и фото лежат в понятных файлах. После правки нужно
**сохранить и «запушить» в GitHub** — сайт обновится сам за пару минут (Vercel).

| Что поменять | Где | Как |
|---|---|---|
| **Телефон, почта, соцсети, адрес, часы работы** | `src/content/contacts.ts` | Найдите нужную строку, поменяйте значение в кавычках |
| **Цены (дневной отдых)** | `src/content/pricing.ts` | Меняете числа |
| **Тексты на сайте** (кнопки, заголовки) | `src/content/translations.ts` | ⚠️ меняйте сразу в трёх языках: `ru`, `uz`, `en` |
| **Фото** | папка `public/images/resort/` | Положите новый файл, затем впишите его в `src/content/images.ts` |
| **Номера (глэмпинг/коттедж)** | `src/content/rooms.ts` | Описание, вместимость, площадь |

**Как сохранить изменения (чтобы попали на сайт):**
```
git add -A
git commit -m "правка: обновил цены"
git push
```
Через 1–2 минуты изменения появятся на chimgandarbaza.uz.

> Если не хотите трогать код — попросите разработчика, у него это займёт минуты.

**Что меняется НЕ в коде, а в кабинете Exely** (secure.exely.com): цены и
доступность номеров, валюта показа (сумы), письма-подтверждения. Это админка
бронирования, не сайт.

## 3. Как передать проект новому разработчику

Дайте новому разработчику три вещи:
1. Ссылку на репозиторий: `https://github.com/Abdurasul99/chimgansoy_resort`
2. Файл **[TECHNICAL_DOCS.md](./TECHNICAL_DOCS.md)** — вся техника.
3. Файл **`ACCESS.md`** — все доступы (он **намеренно не в Git**; передайте его
   безопасно: через менеджер паролей или личным сообщением, не публично).

Плюс пришлите ему `.env.local` (ключи для локального запуска).

Пройдите по каждому пункту — дайте новому исполнителю доступ или смените владельца:

- [ ] **Google Workspace Admin** — почта `manager@chimgandarbaza.uz` и алиасы
- [ ] **Exely** — кабинет бронирования (secure.exely.com), отель `514200`
- [ ] **Resend** — сервис отправки писем (ключ в Vercel)
- [ ] **Telegram-бот** — `@chimgandarbaza_bot` (токен в `.env.local`/Vercel)
